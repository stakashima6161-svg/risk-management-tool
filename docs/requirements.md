# 要件定義メモ

## 背景
TKC九州会会員が、関与先に対して財務諸表の数字をベースにリスクマネジメントの推進を行える入口ツールを作成する。

通常の損害保険会社は、建物・機械・設備などの物的損害や保険商品から入りやすい。一方、TKC会計事務所は、月次試算表、決算書、取引先別管理、損益分岐点、自己資本、現預金、借入返済などの数字を把握しているため、より経営助言に近いリスクマネジメント提案ができる。

## 目指す状態
巡回監査担当者が、関与先に対して次のような会話ができる状態を目指す。

- 主要取引先がなくなった場合、営業利益は赤字になりますか。
- 主要仕入先が止まった場合、在庫で何日持ちますか。
- 売上が30日止まった場合、粗利益はどれくらい失われますか。
- 現預金で固定費と返済を何か月まかなえますか。
- 会社規模に対して自己資本は十分ですか。
- このリスクは、移転・回避・保有・低減のどれで対応しますか。

## 対象利用者
- TKC会員事務所の所長
- 巡回監査担当者
- リスクマネジメント委員会メンバー
- 職員研修で利用する担当者

## 非対象
- 保険証券の補償内容判定
- 保険商品の比較・推奨
- 個別企業名を入力する顧客管理システム
- クラウド上で個人情報を管理する仕組み

## 匿名化方針
入力する情報は、匿名化された財務情報・概算情報に限定する。
関与先名、住所、代表者名、正式な取引先名、正式な仕入先名は入力しない。
